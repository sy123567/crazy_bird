# 减肥追踪应用规格说明

## 1. 项目概述

**项目名称**: WeightLossTracker (减肥追踪器)

**核心功能**: 一款帮助用户追踪减肥进度的移动应用，提供体重记录、进度可视化、数据对比和数据持久化功能。

**目标用户**: 需要减肥或控制体重的人群

## 2. 技术栈

- **框架**: Android Native (Kotlin)
- **UI**: Jetpack Compose + Material Design 3
- **数据库**: Room (数据持久化)
- **图表**: Vico (Compose 图表库)
- **架构**: MVVM + Repository Pattern
- **最小SDK**: 36
- **编译SDK**: 36

## 3. 功能列表

### 3.1 初始化设置
- 设置初始体重（减肥开始时的体重）
- 设置理想体重（目标体重）
- 数据存储在本地数据库

### 3.2 每日进度追踪
- 记录每日当前体重
- 显示当日体重变化（相比前一天）
- 自动计算减肥进度百分比

### 3.3 数据对比
- 相比前一天瘦了多少斤
- 用颜色区分：减少（绿色）、增加（红色）、持平（灰色）
- 清晰的数字显示和箭头指示

### 3.4 统计图表
- 折线统计图显示体重变化趋势
- X轴：日期
- Y轴：体重数值
- 可视化减肥进度

### 3.5 总进度显示
- 已减肥总体重
- 距离理想体重的差距
- 减肥进度百分比
- 完成度可视化进度条

### 3.6 数据持久化
- 所有数据自动保存到本地数据库
- 应用重启后数据不丢失
- 历史记录永久保存

## 4. UI/UX 设计方向

### 4.1 整体视觉风格
- Material Design 3 (Material You)
- 清新、简洁、现代感
- 移动端优先设计

### 4.2 颜色方案
- 主色: 绿色系（代表健康、减肥成功）
- 强调色: 渐变绿色
- 背景: 浅色背景，深色文字
- 数据标识: 绿色（减少）、红色（增加）、灰色（持平）

### 4.3 布局方式
- 单页面布局（Tab导航）
- 顶部：设置区域（初始体重、理想体重）
- 中部：统计图表
- 底部：数据记录列表和总进度

### 4.4 组件设计
- 大号数字显示体重数据
- 卡片式设计展示各项指标
- 圆形进度条显示总体进度
- 简洁的输入界面记录每日体重

## 5. 数据模型

### 5.1 UserSettings (用户设置)
- initialWeight: Float (初始体重)
- targetWeight: Float (目标体重)
- createdAt: Long (创建时间戳)

### 5.2 DailyRecord (每日记录)
- id: Long (主键)
- date: Long (日期时间戳)
- weight: Float (当日体重)
- weightChange: Float (相比前一天变化)
- note: String? (备注，可选)

## 6. 界面结构

```
主界面
├── 顶部设置卡片
│   ├── 初始体重显示
│   ├── 目标体重显示
│   └── 编辑按钮
├── 总体进度卡片
│   ├── 进度百分比
│   ├── 已减肥重量
│   └── 距离目标差距
├── 统计图表区域
│   └── 折线图
├── 今日记录卡片
│   ├── 今日体重输入
│   └── 相比昨日变化
└── 历史记录列表
    └── 每日记录卡片列表
```
